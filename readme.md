# Video-Super-Resolution_Frame-Interpolation

---

## 1. 프로젝트 개요

본 프로젝트는 저화질 영상을 입력으로 받아 프레임 보간을 통해 60fps로 변환하고, 이후 AI 기반 업스케일링을 적용하여 영상을 개선하는 것을 목표로 한다.

본 연구의 핵심은 단순히 화질을 극대화하는 것이 아니라, 프레임 또한 보간해 시각적인 요소를 확보하는것이다.

본 프로젝트는 이론적으로 가장 높은 성능보다는 실제 환경에서 사용 가능한 AI 영상 처리 시스템을 지향한다.

> 여기에 이미지 삽입 권장  
> - 360p 입력 영상과 최종 720p 출력 영상 비교 스크린샷

---

## 2. 데이터 수집 및 모델 확보
[Real_ESRGAN](https://github.com/xinntao/Real-ESRGAN) 업스케일링 모델 \
[vimeo90k](https://cove.thecvf.com/datasets/875) 프레임 보간 모델 학습 데이터

---

## 3. 프레임 보간 기법 (VFI)

프레임 보간 단계에서는 연속된 두 프레임을 입력으로 받아 중간 프레임을 생성하는 CNN 기반 모델을 사용하였다.

해당 모델은 두 프레임을 채널 방향으로 결합한 뒤, 인코더-디코더 구조를 통해 중간 프레임을 예측한다.

이 방식은 복잡한 Optical Flow 기반 모델보다 구조가 단순하며, 처리 속도가 빠르다는 장점을 가진다.

> 여기에 이미지 삽입 권장  
> - 프레임 A, 프레임 B, 생성된 중간 프레임 비교 이미지

---

## 4. 해상도 업스케일링 기법 (Real-ESRGAN)

업스케일링 단계에서는 Real-ESRGAN의 x2 모델을 사용하였다.

초기 실험에서는 x4 모델을 사용하였으나, 처리 시간이 과도하게 증가하여 실시간 또는 준실시간 처리가 어렵다는 문제가 있었다.

이에 따라 본 프로젝트에서는 화질과 처리 속도 간의 균형을 고려하여 x2 업스케일링 방식을 채택하였다.

GPU 메모리를 최대한 활용하기 위해 타일링을 비활성화하고, FP16 연산을 사용하여 처리 속도를 최적화하였다.

> 여기에 이미지 삽입 권장  
> - 업스케일링 전후 프레임 확대 비교 (텍스처 부분 클로즈업)

---

## 5. 처리 순서에 대한 비교 실험

본 프로젝트에서는 다음 두 가지 처리 순서를 비교하였다.

- 프레임 보간 → 업스케일링
- 업스케일링 → 프레임 보간

실험 결과, 프레임 보간을 먼저 수행한 뒤 업스케일링을 적용하는 방식이 전체 처리 시간과 안정성 측면에서 더 효율적인 결과를 보였다.

특히 프레임 보간은 해상도에 따라 연산량이 크게 증가하기 때문에, 저해상도 상태에서 먼저 처리하는 것이 전체 연산 비용을 줄이는 데 효과적이었다.

---

## 6. 실험 환경

본 프로젝트의 실험 환경은 다음과 같다.

- GPU: NVIDIA RTX 4060
- Framework: PyTorch
- Video Processing: OpenCV
- Super Resolution Model: Real-ESRGAN x2
- Precision: FP16

---

## 7. 결론

본 프로젝트를 통해 저해상도 영상의 해상도와 프레임 수를 동시에 개선할 수 있음을 확인하였다.

단순한 보간 방식 대비 AI 기반 접근 방식은 시각적으로 더 자연스러운 결과를 제공하였으며, 처리 속도 또한 현실적인 수준을 유지하였다.

향후에는 프레임 보간 모델의 품질 개선이나 스트리밍 기반 처리 방식을 도입하여 추가적인 성능 향상을 시도할 수 있을 것으로 판단된다.

> 여기에 이미지 삽입 권장  
> - 최종 결과 영상 프레임 캡처
> - 입력 대비 출력 품질 비교 표
